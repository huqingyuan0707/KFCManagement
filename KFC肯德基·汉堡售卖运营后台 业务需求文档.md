## 三、详细业务需求

### 3\.1 工作台总览（首页看板）

**业务定位**：门店运营数据驾驶舱，开门即看核心经营数据

#### 核心功能

- 今日核心KPI展示：今日订单数、今日营收、汉堡销量、待出餐数量、核销完成率

- 实时订单动态：最新10条下单记录滚动展示

- 状态统计卡片：待接单、制作中、待取餐、已完成、已取消订单数量

- 热销榜单：当日TOP10爆款汉堡/套餐排行

- 库存预警提醒：食材不足、临期食材红色高亮提示

- 快捷操作入口：新增商品、核销订单、发布活动、导出日报

### 3\.2 商品管理模块（核心基础）

**业务定位**：统一管理肯德基全品类餐品，以汉堡为核心，包含单品、套餐、小食、饮品

#### 3\.2\.1 商品列表管理

- 支持商品新增、编辑、上下架、批量操作、删除、批量导入导出

- 商品字段：商品名称、分类、售价、会员价、成本价、库存、销量、状态、标签、排序

- 专属标签体系：新品上线、热销爆款、会员专享、限时特惠（匹配UI品牌色徽章）

- 搜索筛选：按名称、分类、上下架状态、标签、时间筛选

#### 3\.2\.2 商品分类管理

支持自定义分类：经典汉堡、香辣汉堡、套餐组合、小食配餐、饮品甜品，支持排序、启用/禁用。

#### 3\.2\.3 规格配置

支持汉堡规格自定义：单层/双层、加芝士、加培根、去洋葱、去酱等可选配置，联动价格与库存。

### 3\.3 订单管理模块（核心业务）

**业务定位**：全渠道订单统一管理，覆盖小程序、外卖平台、线下堂食订单

#### 3\.3\.1 订单列表

- 展示字段：订单编号、下单时间、顾客信息、支付方式、订单金额、商品明细、订单状态、操作

- 订单状态全流程：待接单、制作中、待取餐、已完成、已取消、退款中、退款成功

- 支持按状态、时间、金额、支付方式精准筛选

#### 3\.3\.2 订单操作能力

- 接单、取消接单、一键核销出餐

- 手动退款、部分退款、退款记录查询

- 订单详情查看、订单备注修改、异常订单标记

- 批量导出订单报表、批量标记完成

#### 3\.3\.3 订单售后管理

统一处理用户退款、投诉、异常订单，记录售后原因、处理人、处理时间，形成售后台账。

### 3\.4 后厨出餐模块（门店刚需）

**业务定位**：衔接前台订单与后厨制作，提升出餐效率，杜绝错单漏单

- 实时刷新待制作订单，优先展示新下单、超时未制作订单

## 四、核心业务流程规范

### 4\.1 完整下单出餐流程

用户下单 → 后台实时接收订单 → 前台接单 → 后厨接收制作任务 → 制作完成 → 前台核销出餐 → 订单完结 → 自动扣减库存 → 数据自动统计

### 4\.2 商品上下架流程

新增/编辑商品 → 设置价格、库存、标签 → 保存预览 → 一键上架 → 前台展示售卖 → 售罄自动下架/手动下架

### 4\.3 库存管控流程

食材入库录入 → 售卖自动出库 → 库存低于阈值预警 → 人工盘点补货 → 损耗登记修正数据
